# PWA Features Design

**Goal:** Make FetchBolig installable on mobile/desktop, add offline asset caching, and provide an app update prompt — using `vite-plugin-pwa` with Workbox.

**Approach:** `vite-plugin-pwa` in `generateSW` mode. Config-driven, no hand-written service worker.

**Push notifications:** Deferred to a future iteration.

---

## Architecture

`vite-plugin-pwa` wraps Google's Workbox and handles manifest generation, service worker creation, and caching strategies — all from `vite.config.ts`.

### Caching strategies

| Resource | Strategy | TTL | Rationale |
|----------|----------|-----|-----------|
| Build assets (JS, CSS, HTML) | Precache | Until next deploy | Versioned by Vite hash, always fresh |
| API responses (`/api/*`) | NetworkFirst | 24h fallback | Matches existing localStorage staleness threshold |
| findbolig images (`findbolig.nu`) | CacheFirst | 7 days | Property photos rarely change |

The existing localStorage cache in `appointments.ts` stays as-is. The service worker is an additional layer covering assets and images that localStorage doesn't handle.

## Manifest & Installability

Generated by the plugin from config:

- **name:** "FetchBolig" / **short_name:** "FetchBolig"
- **theme_color:** `#7c3aed` (violet-600, existing UI accent)
- **background_color:** `#ffffff`
- **display:** `standalone`
- **start_url:** `/`
- **icons:** 192x192 + 512x512 PNG, plus maskable variant (user generates via realfavicongenerator.net)

The plugin auto-injects `<link rel="manifest">`, theme-color meta, and apple-touch-icon into `index.html`.

## Update Flow

Use the plugin's `registerSW` virtual module with `prompt` behavior:

1. New service worker detected → toast appears: "New version available" + Refresh button
2. User taps refresh → `updateSW()` → page reloads with new assets
3. No forced auto-reload (avoids interrupting user mid-task)

Hooks into existing `useToastStore` for consistent notification UI.

## Offline Behavior

The app already shows cached appointments via localStorage. The service worker adds:

- Cached app shell (JS/CSS/HTML) loads without network
- Cached property images display offline
- API fallback — if network fails, Workbox returns cached API response

No dedicated offline page needed. Existing "cache stale" warning communicates data freshness.

## Files Changed

| File | Action | Purpose |
|------|--------|---------|
| `client/package.json` | Modify | Add `vite-plugin-pwa` dev dependency |
| `client/vite.config.ts` | Modify | Add PWA plugin config |
| `client/index.html` | Modify | Add apple-touch-icon, theme-color meta |
| `client/src/composables/useAppUpdate.ts` | Create | Composable wrapping `registerSW` for update prompt |
| `client/src/components/AppUpdatePrompt.vue` | Create | Banner for "New version available" |
| `client/src/App.vue` | Modify | Mount AppUpdatePrompt |
| `client/public/` | Add icons | Placeholder icons (replaced by user) |

No server changes. No changes to data layer or auth.
